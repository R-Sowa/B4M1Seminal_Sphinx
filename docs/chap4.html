Sphinxの使い方¶

sphinx-quickstart¶

sphinx-quickstart コマンドを用いると、インタラクティブにいくつかの質問に答えるだけで、ドキュメント作成のために必要なSphinxのプロジェクトを作成することができます。 /YourRepository/Sphinx-files/ に移動して、実際に入力します。

以上の質問に答えると、 /YourRepository/Sphinx-files/ 以下にいくつかのファイルやディレクトリが生成されます。

これらのうち、まずは Makefile と index.rst を編集していきます。簡単にいうと、 Makefile はmacOS (Linux)環境でドキュメントのビルドを行うためのバッチファイル ( make.bat はWindows環境用です。Windows環境の方は、他のページも参考にしてください。)で、 index.rst が、ドキュメントをreSTで実際に作成するファイルになります。x

Makefile を開き、以下のように編集します。強調されている行がデフォルトからの変更行になっています。

 # Minimal makefile for Sphinx documentation
 #

 # You can set these variables from the command line, and also
 # from the environment for the first two.
 SPHINXOPTS    ?=
 SPHINXBUILD   ?= sphinx-build
 SOURCEDIR     = .
 BUILDDIR      = ../docs

 # Put it first so that "make" without argument is like "make help".
 help:
     @$(SPHINXBUILD) -M help "$(SOURCEDIR)" "$(BUILDDIR)" $(SPHINXOPTS) $(O)

 .PHONY: help Makefile

 # Catch-all target: route all unknown targets to Sphinx using the new
 # "make mode" option.  $(O) is meant as a shortcut for $(SPHINXOPTS).
 %: Makefile
     @$(SPHINXBUILD) -b $@ "$(SOURCEDIR)" "$(BUILDDIR)" $(SPHINXOPTS) $(O)

ドキュメントをビルドするディレクトリをGithubの設定に合わせて /YourRepository/docs/ に変更しています。

index.rst¶

以上でドキュメント作成の準備ができました。 index.rst を編集することで、様々なドキュメントが作成できます。編集方法については今回は触れませんが、reSTと呼ばれるマークアップ言語を使用します。今回はGitHub Pagesにウェブサイトとして公開することを目標とし、先に進みます。

現在、 index.rst に何らかの編集を行っていなければ、次のような内容であると思います。

.. Test1 documentation master file, created by
   sphinx-quickstart on Sat Jul  8 22:52:50 2023.
   You can adapt this file completely to your liking, but it should at least
   contain the root `toctree` directive.

Welcome to Test1's documentation!
=================================

.. toctree::
   :maxdepth: 2
   :caption: Contents:



Indices and tables
==================

* :ref:`genindex`
* :ref:`modindex`
* :ref:`search`

一旦こちらの内容を編集せずに、これからビルドを行うことで html ファイルを作成します。

rstファイルのビルド¶

.nojekyll¶

ビルドを行う前に、 /YourRepository/docs/ の直下に .nojekyll というファイルを作成する必要があります。というのは、GitHub Pagesが、 Jekyllというツールを使用しており、このままビルドを行うとJekyllの設定が優先されてしまうため、ページの見た目などが崩れてしまうからです。

touch .nojekyll

.nojekyll ファイルを作成したら、 /YourRepository/Sphinx_files/ に移動して、 make html というコマンドを実行することで、 /YourRepository/docs/ にhtmlファイルなどが作成されます。うまくいっていれば、 open コマンドでhtmlファイルを開くことで、ページが確認できます。

make html
cd ../docs/
open index.html

GitHubへのプッシュ¶

望みのhtmlファイルを作成することができたら、GitHubにプッシュすることで、GitHub Pages上で公開できます。プッシュの具体的な手順については省略しますが、 /YourRepository/docs/ に移動して、全てのファイルをプッシュします。プッシュを行うことができたら、GitHubのリポジトリからGitHub Pagesの欄へと移動すると、画像のように作成したサイトのURLが掲載されているので、URLにアクセスすることでページを確認できます。

注釈

画像では Chapter1 などのコンテンツが存在していますが、これはデフォルトの /YourRepository/docs/ に編集を行っているので、このページの記載通りに行った場合は存在しませんが問題ありません。 GitHub Pagesを用いて、GitHubにSphinxのドキュメントを公開することができました。このページの内容を参考に、実際にドキュメントを作成する際は、 こちら や こちら などの文法が詳しく説明されているウェブサイトを併せて参照して頂けると良いと思います。
